# ADO-1167

## Goal

Remediate `.github/workflows/publish.yml` so tag-triggered NuGet publishing uses Linux + nuget.org Trusted Publishing, removes the broken Windows-only verification and self-mutation flow, and keeps the release process aligned with `CHANGELOG.md`.

## Scope & Changes

### Files changed

- `.github/workflows/publish.yml`
- `CHANGELOG.md`

### Workflow changes

- Moved the publish job from `windows-latest` to `ubuntu-latest`.
- Added `id-token: write` permission for OIDC-based Trusted Publishing.
- Replaced `secrets.NUGET_API_KEY` push flow with `NuGet/login@v1` + temporary API key output.
- Removed the tag-triggered commit/tag self-mutation step entirely.
- Replaced `sn.exe` usage with Linux-compatible assembly metadata checks using PowerShell/.NET reflection.
- Added explicit validation for:
  - changelog version extraction
  - pushed tag matching `v{CHANGELOG_VERSION}`
  - exactly one solution file
  - required signing/login secrets
  - exactly one produced `.nupkg`
  - packaged DLL presence under `lib/`
- Kept trigger scope to `push.tags: ['v*.*.*']`.

### Changelog change

- Updated prerequisite wording to describe Trusted Publishing instead of `NUGET_API_KEY`.
- Clarified that the release build applies the version during CI rather than mutating the project file in-repo.

## Residual Risks

1. GitHub/nuget.org external setup is still required: a nuget.org Trusted Publishing policy must exist for repository `xrmghost/xrmghost-attributes` and workflow file `publish.yml`.
2. `NUGET_PUBLISH_USER` must be configured with the nuget.org profile name expected by `NuGet/login@v1`.
3. `STRONG_NAME_KEY_FULL` must remain present and valid for signed release builds.
4. Local verification could not exercise a real signed publish because repository secrets/OIDC exchange are unavailable outside GitHub Actions.
