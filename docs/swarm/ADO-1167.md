# ADO-1167 — Publish Workflow: Tag-based trigger + nuget.org exclusive

## Goal

Modify `.github/workflows/publish.yml` to use a tag-based release trigger (`v*.*.*`) and publish exclusively to nuget.org. Remove GitHub Packages as a publication target entirely. Add/update `CHANGELOG.md` documenting the v1.0.0 versioning decision. Verify `XrmGhost.Attributes` package ID availability on nuget.org.

---

## Scope & Changes

### `.github/workflows/publish.yml` — modified in place

| Change | Detail |
|--------|--------|
| Trigger replaced | `on: workflow_dispatch` → `on: push: tags: - 'v*.*.*'` |
| `workflow_dispatch.inputs` block removed | `branch` input is no longer needed |
| `ref: ${{ github.event.inputs.branch }}` removed | Tag push auto-checks out the tagged commit |
| `packages: write` permission removed | No longer needed (no GitHub Packages push) |
| Step **"Set NuGet source"** removed | Was configuring `https://nuget.pkg.github.com/…/index.json` |
| Step **"Push to GitHub Packages"** removed | Was `dotnet nuget push … --source "github"` |
| Step **"Push to NuGet.org"** added | `dotnet nuget push ./nupkg/*.nupkg --source https://api.nuget.org/v3/index.json --api-key "${{ secrets.NUGET_API_KEY }}" --skip-duplicate` |
| `git push` in final step updated | Replaced `git push origin ${{ github.event.inputs.branch }}` with `git push origin HEAD` |
| All strong-name signing steps retained | Unchanged |

### `CHANGELOG.md` — updated at repo root

Added a **Versioning Strategy** section before `## [Unreleased]` documenting:
- Publication channel (nuget.org)
- Publication trigger (tag push `v*.*.*`)
- Version source (CHANGELOG.md extraction)
- First public release rationale (restart at `v1.0.0`)
- NUGET_API_KEY prerequisite

## Residual Risks

1. **NUGET_API_KEY secret** — Must be configured in GitHub repository settings before the first `v*.*.*` tag push. The workflow step will emit a non-fatal error if absent (dotnet nuget push will fail with an auth error, not crash the CI runner catastrophically).
2. **Package ID window** — Although the ID was AVAILABLE at the time of this task's execution (2026-05-05), there is a narrow window before the first publish where another author could claim `XrmGhost.Attributes`. Publish the first tag promptly.
3. **Tag-only trigger** — Removing `workflow_dispatch` means the workflow cannot be triggered manually from the GitHub Actions UI. This is intentional per scope.
4. **Version tag vs CHANGELOG sync** — The tag (e.g., `v1.0.0`) must match the first `[x.y.z]` version in CHANGELOG.md. If they diverge, the build will produce a package with the CHANGELOG version, not the tag version.
5. **`git push origin HEAD` in final step** — After a tag-triggered run, `HEAD` points to the checked-out tagged commit. The `git commit` + `git push origin HEAD` sequence pushes the version-bump commit back to the branch that was tagged. This may create a commit that is ahead of the tag; teams should be aware this is an existing design decision, not introduced by this task.
