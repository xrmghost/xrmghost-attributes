# ADO-1164 — XrmGhost.Attributes README Rewrite (Cycle 2)

## Goal

Rewrite `README.md` so the XrmGhost.Attributes package is understandable as a standalone public artifact: what it is, when to use it alone, how it fits in the XrmGhost ecosystem, and an explicit note about the CLI `generate-scenario` dependency. Fix all broken code examples (wrong attribute placement and wrong constructor signatures).

---

## Scope and Modifications

**Only file modified:** `README.md`

No `.cs` source files, `.csproj`, workflows, or CHANGELOG were touched.

### Cycle 1 findings addressed

**F-1 (high / correctness):** README examples placed `InputParameterAttribute`, `PreImageAttribute`, `PostImageAttribute`, and `UnsecureConfigurationAttribute` on *properties*, but all four have `AttributeTargets.Class`. These would produce CS0592 at compile time.

**Fix applied:** Moved all class-only attributes from property level to class level in every affected code example:
- "Basic plugin" example: `[InputParameterAttribute]`, `[PreImageAttribute]`, `[UnsecureConfigurationAttribute]` moved to class level; plain properties remain (no attributes on them).
- "Multiple entities and messages" example: `[InputParameterAttribute]` moved to class level.
- "Column-filtered entity images" example: `[InputParameterAttribute]`, `[PreImageAttribute]`, `[PostImageAttribute]` moved to class level.
- "Stage, mode, and user impersonation" example: `[InputParameterAttribute]` moved to class level.
- "Multiple solutions" example: `[InputParameterAttribute]` moved to class level.
- "Secure and unsecure configuration" example: already correct (class level) — unchanged.

### Cycle 1 work retained (all scope items from cycle 1 preserved)

- Standalone Use section explaining zero-runtime-overhead usage without the framework.
- XrmGhost Ecosystem section covering `xrmghost-framework-host` and `xrmghost-cli generate-scenario`.
- Explicit CLI note: `generate-scenario` degrades with a warning if attributes are absent.
- Installation section with GitHub Packages NuGet source configuration and `dotnet add package` command.
- Current package version `0.6.27` displayed.
- Stale "generated as part of migration" footer removed.
- All constructor signatures verified against source `.cs` files.
- No internal references (no ADO URLs, no private infrastructure).
- Added clarification note in Included Attributes table: "All attributes target `AttributeTargets.Class` — they must be placed on the plugin class, not on individual members."
