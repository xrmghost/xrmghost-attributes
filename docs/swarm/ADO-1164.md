# ADO-1164 — XrmGhost.Attributes README rewrite

## Goal

Rewrite `README.md` so the XrmGhost.Attributes package is understandable as a standalone public artifact: what it is, when to use it alone, how it fits in the XrmGhost ecosystem, and an explicit note about the CLI `generate-scenario` dependency. Fix all broken single-argument constructor examples that would not compile.

---

## Scope and Modifications

**Only file modified:** `README.md`

No `.cs` source files, `.csproj`, workflows, or CHANGELOG were touched.

### Changes made

1. **Fixed broken constructor usages:**
   - `PreImageAttribute` requires 2 args `(name, entityJson)` — all `[PreImage("primary")]` single-arg usages replaced with correct 2-arg form using realistic placeholder JSON.
   - `PostImageAttribute` requires 2 args `(name, entityJson)` — same fix.
   - `InputParameterAttribute` requires 2 args `(name, valueJson)` — all `[InputParameter("Target")]` single-arg usages replaced with correct 2-arg form.
   - `UnsecureConfigurationAttribute` requires 1 mandatory arg `(configuration)` — `[UnsecureConfiguration]` no-arg usage replaced with `[UnsecureConfigurationAttribute("{ \"featureFlag\": true }")]`.

2. **Added "Standalone Use" section** — explains using the package without any other XrmGhost component; lists typical standalone use cases (custom tooling, documentation generation, test harnesses).

3. **Added "XrmGhost Ecosystem" section** — explains:
   - `xrmghost-framework-host` reads attributes via reflection to auto-populate plugin properties before `Execute`.
   - `xrmghost-cli generate-scenario` uses attribute values to seed test scenario files.
   - Explicit note: `generate-scenario` degrades with a warning if attributes are absent.

4. **Added "Installation" section** — shows GitHub Packages NuGet source configuration (`dotnet nuget add source`) and `dotnet add package XrmGhost.Attributes --version 0.6.27`.

5. **Included current package version** `0.6.27` (from `XrmGhost.Attributes.csproj`).

6. **Removed stale italic footer** — `*This README was generated as part of the project's migration to a standalone NuGet package.*` is gone.

7. **Restructured for public audience** — added Table of Contents, converted attribute list to a table with Target/Description columns.

8. **No internal references** — no ADO URLs, no internal team names, no private infrastructure.

## Risks Residual

- `UnsecureConfigurationAttribute` and `SecureConfigurationAttribute` are applied at class level in the source (`AttributeTargets.Class`) but the original README showed them on properties. The new README applies them at class level in all examples — matching the actual `AttributeUsage`. The framework-host presumably maps the configuration value to a property some other way (not via property-level attribute). This is consistent with the `.cs` source.
- Version `0.6.27` is hardcoded in the README based on `XrmGhost.Attributes.csproj`. It will become stale when the version is bumped. Acceptable for now — standard README practice.
