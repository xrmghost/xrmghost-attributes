# ADO-1166 — XrmGhost.Attributes.Tests: xUnit Smoke Tests

## Goal

Create a new xUnit test project `XrmGhost.Attributes.Tests` in the `xrmghost-attributes` solution with smoke tests for all 10 attribute types. Tests verify instantiation and property values without any Dataverse runtime dependency.

## Scope & Modifications

### New files created

| File | Description |
|------|-------------|
| `XrmGhost.Attributes.Tests/XrmGhost.Attributes.Tests.csproj` | Test project targeting `net8.0` with xunit, xunit.runner.visualstudio, Microsoft.NET.Test.Sdk |
| `XrmGhost.Attributes.Tests/AttributeTests.cs` | All 10 test classes (15 total `[Fact]` tests) |
| `Directory.Build.props` | Empty root MSBuild props (placeholder, actual exclusion in targets) |
| `Directory.Build.targets` | Excludes `XrmGhost.Attributes.Tests/**` from main project's `Compile` glob when `IsTestProject != 'true'` |

### Modified files

| File | Change |
|------|--------|
| `xrmghost-attributes.sln` | Added `XrmGhost.Attributes.Tests` project entry via `dotnet sln add` |

### Files NOT modified

All existing `.cs` attribute files and `XrmGhost.Attributes.csproj` were left untouched per constraints.

## MSBuild Glob Fix

The .NET SDK `XrmGhost.Attributes.csproj` uses `netstandard2.0` with `LangVersion=8.0` and its default `**/*.cs` glob recursively included the new test files. Since the constraint prohibits modifying the main `.csproj`, a `Directory.Build.targets` (evaluated _after_ SDK globs) was added at the repo root with a conditional `<Compile Remove>` that excludes `XrmGhost.Attributes.Tests/**` only when `IsTestProject != 'true'`.

## Residual Risks

- `Directory.Build.targets` is a new file at repo root. If future projects are added to this solution that also have subdirectory `.cs` file placement issues, the same mechanism will protect them automatically. No risk to existing behaviour.
- The `Directory.Build.props` file (empty content) is benign but present. It was created initially with content and then superseded by `Directory.Build.targets`.

## Notes for Review

- The `Directory.Build.targets` trick (not `.props`) is required because SDK default globs are resolved in `.props` phase; a `Remove` in `.targets` correctly fires after the glob has populated `Compile`.
- The `IsTestProject = true` flag is set in the test `.csproj`; the condition `'$(IsTestProject)' != 'true'` correctly exempts the test project itself from the Remove (so the test project keeps all its own `.cs` files).
- No `LangVersion` was specified in the test csproj — it inherits the default for `net8.0` which is C# 12, hence block-scoped namespaces were used for compatibility should the file ever be compiled under the parent project in edge cases.
